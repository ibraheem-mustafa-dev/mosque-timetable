# Changelog

All notable changes to this project will be documented in this file.

## [3.3.0] — 2026-03-03

### Added
- **Ramadan Mode — Suhoor column in main timetable**: When Ramadan dates are configured in Settings, a tinted "Suhoor" column is automatically inserted between Fajr and Sunrise showing the suhoor end time (Fajr − configurable margin, default 15 min) for every day of the month.
- **Ramadan Mode — Auto-injected banner above timetable**: When Ramadan is active, the `[ramadan_info layout="banner"]` banner auto-renders above the main prayer table showing today's day number, Suhoor end time, Iftar time (Maghrib), and a live countdown to Iftar.
- **Ramadan Mode — Suhoor in mobile prayer cards**: Mobile card layout now shows a styled Suhoor time item (tinted, accent border) after Fajr, only during Ramadan.
- **CSS — `.suhoor-item` mobile card style**: Accent left-border and background tint distinguishes Suhoor from other prayer times in mobile view.

## [Unreleased]

### Added
- **Task A: Per-month PDF upload** - Admins can now upload print-ready PDFs for each month in the Timetable admin page. When a PDF exists, visitors see a "Download Timetable" button; otherwise they see a "Print Timetable" button that triggers window.print(). Supports both ACF Pro and options fallback storage modes.

- **Task B: Visitor export modal with options and export endpoint** - Replaced the simple export button with a comprehensive modal that allows visitors to customize their calendar export. Features include: date range selection (full year or selected month), Jamāʿah time inclusion toggle, multiple notification options (Start, 5m, 10m, 20m, 30m), Jummah service selection (both, 1st, or 2nd), and sunrise warning alarms. New GET/POST REST endpoint at `/wp-json/mosque/v1/export-ics` supports all options with backward compatibility. Modal uses CSS variables for theme integration and includes Google Calendar import instructions.

- **Task C: Subscribe button with optional override** - Enhanced the existing "Subscribe to Calendar" button with an optional custom URL override feature. Admins can now set a custom subscription URL in the Mosque Configuration settings to point to external calendar feeds if needed. When no custom URL is set, the button continues to use the default `/prayer-times/calendar.ics` feed. The feature works in both ACF Pro and options fallback modes. New helper function `mt_get_subscribe_url()` provides a centralized way to retrieve the appropriate subscription URL throughout the plugin.

- **Task D: Friday/Jummah frontend behaviour** - Updated frontend timetable display to properly handle Friday/Jummah prayers. On Fridays, the Zuhr/Jummah column now shows both Jummah times separated by "/" (e.g., "1:15 / 2:30") instead of displaying Zuhr start and Jamāʿah times. The table header continues to read "Zuhr / Jummah" as specified. Updated both the main timetable shortcode and today's prayers widget. ICS export logic was already correctly excluding Zuhr on Fridays and including appropriate Jummah times based on visitor selection.

- **Task E: Mobile Pattern A transform and sticky Next Prayer bar** - Implemented comprehensive mobile optimization with Pattern A transformation. On screens ≤ 480px, the table converts to elegant cards showing date, Hijri date, and prayer times with Jamāʿah times stacked under start times. Added a sticky prayer bar at the top displaying today's date (Gregorian and Hijri) with swipeable prayer chips that auto-center on the next prayer. Desktop shows all chips inline without scrolling. Includes full keyboard navigation (arrow keys, Home/End, Enter/Space), ARIA accessibility (tablist, aria-selected), and smooth scrolling to corresponding table rows/cards. Fully responsive with no horizontal overflow on mobile.

- **Task F: Terminology overrides** - Added comprehensive terminology customization system allowing admins to override labels throughout the plugin interface. Admins can define custom terminology mappings (e.g., "Mosque" → "Masjid", "Zuhr" → "Dhuhr", "Maghrib" → "Maghreb") in the Configuration settings. Each override can be enabled/disabled individually. Changes apply to all frontend displays (table headers, mobile cards, prayer chips, today's prayers widget) and admin menu titles. The system is case-sensitive and applies only to UI labels, not internal data or field names. Works in both ACF Pro and options fallback modes with dynamic add/remove functionality.

- **Task G: SEO discovery and structured data** - Enhanced SEO optimization with comprehensive search engine and AI tool support. Added XML sitemap at `/prayer-times-sitemap.xml` containing prayer times pages and archive URLs with proper lastmod dates and priority values. Implemented `/llms.txt` metadata file following the LLMs.txt specification for AI tool discovery, providing structured information about the plugin's purpose and data sources. Added robots.txt integration to advertise both the sitemap and llms.txt file. Enhanced structured data with Schema.org markup including Organization (mosque details with contact info and location), WebSite (with SearchAction for prayer times searches), and Dataset schemas (describing prayer times data with temporal coverage and geographic scope). All SEO features integrate seamlessly with existing WordPress architecture and respect user privacy settings.

- **Task I: Multi-year archive pages and default year handling** - Implemented comprehensive multi-year archive functionality with dedicated pages for browsing historical prayer times. Added `/prayer-times/` main archive page listing all available years with visual indicators for current vs. historical data. Created `/prayer-times/{year}/` year-specific archive pages showing monthly breakdowns with availability status and direct links to monthly timetables. Enhanced default year management with intelligent validation, auto-advancement logic that detects when new year data is available, and robust fallback handling. Archive pages feature responsive design with professional styling, clear navigation between years and months, and integration with existing terminology overrides. Updated XML sitemap to include all archive URLs for improved SEO discoverability. The system automatically advances to new years when sufficient data is detected, ensuring users always see current information while maintaining access to historical archives.

- **Task J: Web push notifications (opt-in, privacy-safe)** - Implemented comprehensive push notification system following best-practice permission UX. Added "Prayer Reminders" button to prayer bar and timetable pages that shows pre-permission modal explaining benefits before requesting native permission. Admin settings include VAPID public/private key configuration, default reminder offset options (5-30 minutes), sunrise warning toggle with configurable offset, and editable privacy note text. Backend features WP-Cron job (1-minute cadence) using minishlink/web-push library to send timely notifications for each subscription's selected reminder offsets. REST endpoints at `/wp-json/mosque/v1/subscribe` and `/wp-json/mosque/v1/unsubscribe` handle subscription management with proper nonce and capability checks. Respects Friday rules (no Zuhr notifications; Jummah 1/2 as configured), handles timezone properly, includes graceful error handling for expired subscriptions, and provides clear unsubscribe functionality. Never prompts on page load - only after explicit user action. Works without breaking ACF/Options fallback mode. Service worker integration supports notification clicks and offline functionality.

- **Task K: Translation and internationalization** - Implemented complete translation readiness with WordPress i18n best practices. Added text domain loading in plugin initialization, wrapped all user-facing strings in `__()` and `_e()` translation functions including admin interface labels, error messages, and frontend display text. Created comprehensive `languages/mosque-timetable.pot` template file containing all translatable strings with proper context markers and placeholders for variable content using `sprintf()`. Enhanced date localization using `date_i18n()` for user-facing date displays. Added comprehensive RTL (Right-to-Left) language support with CSS rules targeting `html[dir="rtl"]` for Arabic, Hebrew, Urdu, Persian and other RTL scripts, including automatic language detection and HTML dir attribute injection. RTL support covers all components: prayer tables, modal dialogs, navigation tabs, prayer cards, button layouts, and responsive mobile patterns. Includes font family recommendations for non-Latin scripts. Plugin is now fully ready for community translation into any language with proper text directionality support.
