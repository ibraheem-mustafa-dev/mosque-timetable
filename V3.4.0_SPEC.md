# Mosque Timetable v3.4.0 - Rebuild Spec

## Core Features (Ramadan + UX)

### 1. Hijri Auto-Detection
- `auto_detect_ramadan_dates($year)` using Kuwaiti/tabular algorithm
- `gregorian_to_hijri()` and `hijri_to_gregorian()` helper methods
- Admin start/end date fields become optional overrides (algo provides defaults)
- Results cached in 24hr transient
- Add to `class-mosque-timetable.php`

### 2. Dual Countdown (Iftar -> Suhoor)
- After Iftar time passes, countdown switches to "Until Suhoor" (tomorrow's)
- PHP: new `get_tomorrow_prayer_data()` method outputs `data-suhoor-next` attribute
- JS: rewrite `updateRamadanCountdowns()` to check current time vs Iftar, swap target

### 3. Last 10 Nights
- `is_last_ten_nights()` + `get_ramadan_day_number()` PHP helpers
- Visual treatment: glowing badge on banner, deeper navy banner theme, gold border
- CSS animation `mt-glow` keyframe
- Table rows in last 10 get `.mt-last-ten` class

### 4. Taraweeh Time
- New admin field: `ramadan_taraweeh_time`
- Display in banner, card, and compact layouts when set

### 5. Full i18n
- All Ramadan strings wrapped in `__()` / `_e()` / `esc_html_e()` with `mosque-timetable` text domain

## Widget UX & Bug Fixes (March 3 Feedback)

### 6. JS Timer Bug Fix (Dynamic Next Prayer)
- Current issue: Timer gets stuck at `00:00` when the prayer arrives and requires page reload.
- Fix: Pass the day's full prayer schedule to JS (via `wp_localize_script` or `data-schedule` attribute). When `diff <= 0`, JS automatically finds the next chronological prayer, updates the target, and updates the UI label (e.g., "Dhuhr in" -> "Asr in").

### 7. "Dhuhr" Default Spelling
- Update the default spelling in the plugin from "Zuhr" to "Dhuhr" to match the user request.

### 8. UI Polish: Vertical Alignment & Font
- Ensure prayer names in the widget vertically align perfectly with the times and countdowns.
- Fix font loading (inherit correctly or load the designated font).

### 9. Calls to Action (Clickable Widget)
- Wrap the widget in a link or add explicit CTA buttons so clicking it sends the user to the main timetable page (configurable via settings or shortcode attribute).

### 10. Desktop Widget Sizing
- Introduce a larger, more prominent layout option for desktop displays (a non-compact, sleek horizontal bar or block).
